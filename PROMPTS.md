# 常用调研指令模板

本文件收集当前自动化调研 skills 库的常用调用方式。使用时将主题替换为你的研究领域即可。

## 总控入口

```text
使用 $auto-research 调研“空间音频大模型”最近一周的研究和产业动态，生成中文 HTML 报告，并更新本地知识库。
```

```text
使用 $auto-research 调研“AI Agent 评测”，自动判断适合的调研粒度，输出中文 HTML 报告。
```

## 候选来源采集增强

适合希望先用无密钥公共 API 建立候选来源池，再做人工/Agent 复核的调研。

```text
使用 $auto-research 调研“AI Agent 评测”最近一周动态。请先用内置候选来源采集脚本检索 arXiv、OpenAlex 和 GitHub，再补充官方博客/新闻来源，最后生成中文 HTML 报告并更新本地知识库。
```

```text
使用 $research-monthly 调研“多模态智能体”最近一个月进展。要求先生成候选来源 JSONL，复核后只引用能支撑结论的论文、仓库和官方来源。
```

## 日调研

适合查询今天或最近 24 小时的最新动态；默认不包含研究热词和趋势聚类。

```text
使用 $research-daily 查询“视频生成模型评测”今天最新动态，要求列出来源、时间和潜在影响。
```

```text
使用 $research-daily 调研“OpenAI API”最近 24 小时变化，区分官方更新、社区信号和待验证消息。
```

## 周调研

适合生成领域周报，覆盖最近 7 天论文、项目、产品和产业动态；默认不包含研究热词和趋势聚类。

```text
使用 $research-weekly 调研“AI Agent 评测”最近一周动态，输出中文 HTML 周报，并更新 knowledge_base。
```

```text
使用 $research-weekly 生成“空间音频大模型”最近一周调研报告，按论文、开源项目、产业动态和风险分类。
```

周调研和日调研会默认对区间内所有文献执行 trace，并把分析作为可折叠区块嵌入 HTML。

## 单篇论文技术溯源

适合碰到一篇论文后，希望直接获得 AI 辅助阅读、技术脉络和复现风险分析。

```text
使用 $paper-trace 分析 arXiv:2606.13095，生成中文 HTML，重点说明它继承了哪些前置技术、相对前作的差异和复现风险。
```

```text
使用 $paper-trace 对论文 “Balancing ASR and diarization in end-to-end LLMs for multi-talker speech recognition” 做技术溯源分析，并给出后续追踪 query。
```

## 月调研

适合做阶段性综述，覆盖最近 30 天的系统性变化。

```text
使用 $research-monthly 调研“端侧多模态大模型”最近一个月论文、项目和产业动态，生成中文 HTML 报告。
```

```text
使用 $research-monthly 复盘“多智能体协作框架”最近一个月进展，总结核心趋势、代表项目和下月追踪问题。
```

## 近一年研究热词

适合提取领域关键词、爆发词、方法名、数据集名和评测方向。

```text
使用 $research-yearly-hotwords 分析“语音大模型”近一年研究热词，按证据强度和增长信号排序。
```

```text
使用 $research-yearly-hotwords 分析“自动驾驶世界模型”近一年热词，要求给出每个热词的含义、代表来源和研究启发。
```

## 近一年研究趋势

适合做战略判断、选题规划和研究路线分析。

```text
使用 $research-yearly-trends 分析“具身智能大模型”近一年研究趋势、机会和风险，生成中文 HTML 趋势报告。
```

```text
使用 $research-yearly-trends 调研“世界模型”近一年发展方向，区分真实趋势、短期热点和证据不足的推测。
```

## 组合调研

适合先建立长期背景，再追踪短期变化。

```text
使用 $auto-research 对“音频语言模型”执行两阶段调研：先分析近一年研究趋势，再生成最近一周动态周报。两个报告都输出 HTML，并更新本地知识库。
```

```text
使用 $auto-research 调研“多模态推理模型”：生成近一年研究热词分析和最近一个月动态报告，最后给出后续选题建议。
```

## 输出要求增强

可追加到任意指令后面：

```text
报告需要包含：核心判断、代表来源、风险与限制、下一步查询建议，并确保每个事实性判断都有来源链接。
```

```text
报告面向科研选题，请重点分析可做的研究空白、数据/评测瓶颈和未来 3 个月值得跟踪的问题。
```

```text
报告面向工程落地，请重点分析可复现项目、产品动态、部署风险和短期可验证机会。
```
